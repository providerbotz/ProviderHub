📋 Manual Setup Steps (Optional)
1 Enable Firebase Authentication 2 Create Admin Account 3 Register Admin in Database 4 Set Database Security Rules 5 Test Your Login

Step 1: Enable Firebase Authentication

1

Go to Firebase Console

Open your browser and go to console.firebase.google.com. Select your project nhbo-90d76.

2

Navigate to Authentication

In the left sidebar, click on Build → Authentication. If this is your first time, click Get Started.

3

Enable Email/Password Provider

Click on the Sign-in method tab. Click on Email/Password provider. Toggle the Enable switch to ON. Click Save.

Success

Email/Password authentication is now enabled for your Firebase project!

Step 2: Create Admin Account

1

Go to Users Tab

In Authentication section, click on the Users tab.

2

Add New User

Click the Add user button. Enter your admin email and a strong password. Click Add user.

Important

Use a strong password with at least 8 characters including uppercase, lowercase, numbers, and special characters. Save this password securely - you'll need it to login!

3

Copy User UID

After creating the user, you'll see a table with user details. Copy the User UID (a long string like abc123xyz456...). Paste it in the generator above!

Step 3: Register Admin in Database

1

Use the Automatic Generator

The easiest way is to use the Automatic Setup Generator at the top of this page. Just paste your UID and email, then click Push to Firebase.

2

Or Add Manually

Go to Firebase Console → Realtime Database → Click on database URL → Add data:

adminUsers └── [your-user-uid] ├── email: "your-admin@email.com" ├── role: "admin" └── createdAt: [timestamp]

Step 4: Set Database Security Rules

1

Go to Rules Tab

In Realtime Database, click on the Rules tab.

2

Copy Rules from Generator

Use the Automatic Generator above to get the security rules, or copy the rules below:

{ "rules": { "adminUsers": { ".read": "auth != null && root.child('adminUsers').child(auth.uid).exists()", ".write": "auth != null && root.child('adminUsers').child(auth.uid).exists()" }, "movies": { ".read": true, ".write": "auth != null && root.child('adminUsers').child(auth.uid).exists()" }, "categories": { ".read": true, ".write": "auth != null && root.child('adminUsers').child(auth.uid).exists()" }, "users": { ".read": true, ".write": true }, "subscriptions": { ".read": true, ".write": "auth != null && root.child('adminUsers').child(auth.uid).exists()" }, "paymentRequests": { ".read": true, ".write": true }, "settings": { ".read": true, ".write": "auth != null && root.child('adminUsers').child(auth.uid).exists()" }, "comments": { ".read": true, ".write": true } } }
3

Publish Rules

Click the Publish button to save your rules.

Security Rules Explained

• adminUsers: Only existing admins can read/write.
• movies, categories, settings: Admins can write, everyone can read.
• users: Public read/write for user signup & profile updates.
• subscriptions: Admins can manage, everyone can view plans.
• paymentRequests: Public read/write for payment submissions.
• comments: Public read/write for all users.

Step 5: Test Your Login

1

Go to Admin Panel

Open admin.html in your browser.

2

Enter Credentials

Enter the email and password you created in Step 2. Click Login.

3

Verify Access

If everything is set up correctly, you should see the Admin Dashboard. Your email will be displayed in the sidebar.

Congratulations!

Your admin panel is now secured with Firebase Authentication! Only users registered in the adminUsers database node can access the admin panel.

Troubleshooting

?

"Access denied" Error

Your user is not in the adminUsers node. Use the generator above to add your UID to the database.

?

"User not found" Error

The email doesn't exist in Firebase Authentication. Create the user first in Firebase Console → Authentication → Users.

?

"Permission denied" when saving data

Your database rules may not be set correctly. Make sure you published the security rules from Step 4.

?

Generator "Push to Firebase" not working

The database rules might be blocking writes. Temporarily use open rules for setup:

{ "rules": { ".read": true, ".write": true } }
Important

After setup, immediately replace with secure rules!
